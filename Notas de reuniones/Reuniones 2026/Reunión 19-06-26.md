## 3. Estrategia de Fine-Tuning (Ajuste Fino)

Para el entrenamiento y optimización del modelo, se estructuran los siguientes requisitos y flujos de trabajo:

### Requisitos del Dataset

**Estructura de Pares:** Cada entrada debe componerse de:
* **Input:** Texto plano/bruto (por ejemplo, el output directo del OCR).
* **Output:** Estructura limpia y formateada (ej. un objeto **JSON** con los metadatos clave extraídos).
* **Fase de Curación:** Es obligatorio realizar una **limpieza previa de datos** sobre el texto de entrada para eliminar ruido transaccional o errores de digitalización.

### Arquitectura de Validación

1. **Entorno Activo:** Se asume la preexistencia de un modelo base corriendo en un servidor/entorno local.
2. **Conjunto de Control:** Disponer de un volumen de datos ya validados y controlados.
3. **Dataset de Testing:** Separar un subconjunto de datos reservados exclusivamente para evaluar las métricas de precisión del modelo tras el fine-tuning, garantizando que no haya sesgo en las pruebas.
